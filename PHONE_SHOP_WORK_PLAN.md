# PhoneShop Work Plan

최종 갱신: 2026-04-12

## 1. 목표

- 휴대폰 매장 운영에 필요한 재고, 고객, 판매, 미수금, 정책 관리 흐름을 MVP 범위로 완성한다.
- 관리자와 직원이 같은 워크스페이스를 쓰되, 권한에 따라 접근 가능한 메뉴를 분리한다.
- 대시보드와 기본 보고서로 현장 운영 숫자를 바로 확인할 수 있게 한다.

## 2. 현재 상태 요약

- [x] 로그인, 권한 분리, 보호 라우트
- [x] 기초정보/정책 CRUD
- [x] 재고 관리
- [x] 고객 관리
- [x] 판매 등록/취소와 계산 로직
- [x] 미수금 목록/수납 등록/수납 취소
- [x] 대시보드 실데이터, CSV 다운로드, 인쇄용 보고서
- [ ] 안정화, 테스트 확장, 배포 준비

## 3. 단계별 계획

### 1단계. 데이터 구조와 저장 계층

목표:
- SQLite + Prisma 기반 데이터 계층을 세운다.

체크리스트:
- [x] Prisma 도입과 SQLite 연결
- [x] 사용자, 고객, 재고, 판매, 미수금, 수납, 정책 스키마 설계
- [x] 마이그레이션 생성
- [x] 시드 데이터 구성

완료 기준:
- 로컬 DB가 마이그레이션과 시드로 재구성된다.

### 2단계. 인증과 권한

목표:
- 관리자/직원 로그인과 메뉴 접근 제어를 붙인다.

체크리스트:
- [x] credentials 로그인
- [x] 세션 쿠키 기반 인증
- [x] 보호 라우트 적용
- [x] 관리자 전용 메뉴 제한

완료 기준:
- 로그인 없이 워크스페이스에 접근할 수 없고, 직원은 설정 메뉴에 접근할 수 없다.

### 3단계. 기초정보와 정책 관리

목표:
- 통신사, 요금제, 부가서비스, 정책 CRUD를 실데이터로 관리한다.

체크리스트:
- [x] 기초정보 CRUD 연결
- [x] 정책 CRUD 연결
- [x] 활성 상태/기간 기준 조회
- [x] 판매 화면 참조 데이터 연결

완료 기준:
- 판매에 필요한 기준 데이터가 설정 화면에서 관리된다.

### 4단계. 재고 관리

목표:
- IMEI 기준 재고 등록, 검색, 상태 관리 흐름을 완성한다.

체크리스트:
- [x] 재고 등록/수정
- [x] IMEI 중복 방지
- [x] 상태/담당자/노출 필터
- [x] 판매 가능 재고만 판매 화면에 노출

완료 기준:
- 재고와 판매 가능 상태가 실제 DB 기준으로 관리된다.

### 5단계. 고객 관리

목표:
- 고객 등록, 검색, 상세 이력 조회를 완성한다.

체크리스트:
- [x] 고객 등록/수정
- [x] 연락처 정규화와 중복 방지
- [x] 고객 검색/필터
- [x] 판매 이력과 미수금 이력 연결

완료 기준:
- 고객 페이지에서 기본 정보와 거래 이력을 함께 본다.

### 6단계. 판매 관리와 계산 로직

목표:
- 판매 등록, 정책 자동 매칭, 수익 계산, 취소 흐름을 구현한다.

체크리스트:
- [x] 판매 등록 서버 액션
- [x] 할인/리베이트/정책 수익 계산
- [x] 재고 상태 전환과 고객 현재 통신사 갱신
- [x] 판매 취소와 관련 데이터 복구

완료 기준:
- 판매 등록 시 최종 판매가, 실제 수납, 미수금, 총이익이 계산된다.

### 7단계. 미수금과 수납

목표:
- 미수금 목록과 수납 처리 흐름을 구현한다.

체크리스트:
- [x] 실데이터 기반 미수금 목록
- [x] 고객/기간/상태 필터
- [x] 수납 등록
- [x] 수납 취소
- [x] 초과 수납 차단과 잔액 재계산

완료 기준:
- 부분 수납과 완납 상태가 서버에서 다시 계산된다.

### 8단계. 대시보드와 기본 보고

목표:
- 현장에서 바로 보는 핵심 숫자와 기본 보고 기능을 붙인다.

체크리스트:
- [x] 대시보드 실데이터 연결
- [x] 오늘 판매 건수 / 오늘 수납 금액 / 현재 미수금 잔액 표시
- [x] 선택 기간 리베이트 / 정책 수익 / 총이익 표시
- [x] 기간 프리셋과 직접 지정 필터 적용
- [x] CSV 다운로드 1차 구현
- [x] 인쇄용 보고서 페이지 1차 구현

완료 기준:
- 홈 대시보드 카드가 판매/수납/미수금 실데이터를 보여준다.
- 동일한 집계 결과를 CSV와 인쇄용 보고서에서 다시 확인할 수 있다.

### 9단계. 안정화와 출시 준비

목표:
- MVP를 실제 사용 가능한 상태로 다듬고 배포 준비를 마친다.

체크리스트:
- [ ] 판매 / 수납 / 계산 로직 단위 테스트 보강
- [ ] 대시보드 / 보고 / 권한 흐름 E2E 확장
- [ ] 예외 처리와 빈 상태 문구 정리
- [ ] 문서 최종 정리
- [ ] GitHub / Vercel 연결과 초기 배포 준비

완료 기준:
- 핵심 흐름이 자동 테스트로 검증되고, 배포 가능한 상태가 된다.
